# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

Двуязычный (армянский + русский) веб-тест по Конституции Республики Армения для подготовки к экзамену на гражданство. Чистый статический сайт: HTML + CSS + ванильный JS, **без сборки, без npm, без фреймворков, без бэкенда**. Открывается как обычные файлы.

## Запуск

Локальный сервер (нужен именно сервер, а не `file://`, потому что страницы подключают внешние `.js`):

```bash
python3 -m http.server 8080
# затем открыть http://localhost:8080/ («Тест на знание Конституции», 33 вопроса)
# и http://localhost:8080/full.html («Тест для подготовки», 226 вопросов)
```

Тестов/линтеров нет. Проверка — вручную в браузере (см. «Чек-лист поведения» ниже).

## Архитектура

Две страницы используют **одну общую логику и стили**, отличаясь только данными и разметкой шапки:

| Страница | Заголовок | Подключает | Банк вопросов |
|---|---|---|---|
| `index.html` | «Тест на знание Конституции Республики Армения (arlis.am)» | styles.css, articles.js, **questions-official.js**, config.js, app.js | 33 official |
| `full.html` | «Тест для подготовки» | styles.css, articles.js, **questions-full.js**, config.js, app.js | 226 = 33 official + 193 generated |

Порядок подключения скриптов важен: данные (`articles.js`, `questions-*.js`) и `config.js` грузятся **до** `app.js`, который читает их через глобальные `const`.

**Контракт между файлами (всё через глобальные переменные, без модулей):**
- `articles.js` → `const ARTICLES` (карта `"номер_статьи" → русский текст`, 165 ключей по редакции Конституции РА 2015) и `const CONSTITUTION_URL`.
- `questions-official.js` / `questions-full.js` → `const ORIGINAL_QUESTIONS` (массив). Каждая страница грузит ровно один из них; имя переменной одинаковое, поэтому `app.js` не знает, какой тест запущен.
- `config.js` → `const TEST_COUNTS = { official: 33, full: 226 }` и `const LAST_UPDATED` (дата). Нужен для кросс-ссылки (промо-кнопка ведёт на *другой* тест, чьи данные на странице не загружены) и для даты в футере. Число вопросов **текущего** теста при этом всегда берётся из `ORIGINAL_QUESTIONS.length`, а не из `TEST_COUNTS`.
- `app.js` читает `ORIGINAL_QUESTIONS`, `ARTICLES`, `TEST_COUNTS`, `LAST_UPDATED`, рендерит в `#quiz`.

**Форма вопроса:**
```js
{ hy: "...", ru: "...",
  options: [ {hy,ru}, {hy,ru}, {hy,ru} ],
  correct: 0|1|2,            // индекс правильного варианта
  source: "official"|"generated",  // ровно два значения; тег "appeared" НЕ используется
  article: "47",             // ключ в ARTICLES; опционален
  note: "..." }              // служебная пометка происхождения; в UI не показывается
```

`source` принимает только `"official"` (33 вопроса из постановления № 1040-Н) и `"generated"` (193 вопроса, составленные по тексту Конституции РА 2015). Старого пула из 142 generated больше нет. Других значений source (например `"appeared"`) в банке нет и быть не должно.

## Ключевые механизмы в app.js (поведение, которое нельзя ломать)

- **Два режима, независимое состояние.** `modeData.learn` и `modeData.test` — у каждого свой массив `questions` и свой `state` (answers/score/answered). Глобальные `questions`/`state` — ссылки на текущий режим.
- **Режим «Обучение»** = все вопросы (перетасованные). **Режим «Самопроверка»** = `TEST_QUESTION_COUNT` (33) случайных из всего банка (`buildTestQuestions` = shuffle + slice).
- **Бейджи источника автоматические:** `SHOW_BADGES = (число разных source) > 1`. `index.html` (всё `official`) → бейджей нет; `full.html` (смешанный) → два бейджа: «Պաշտоնական · Официальный» и «Գեներացված · Сгенерирован». Третьего бейджа нет (см. про `source` выше). Не хардкодить per-page.
- **Число вопросов автоматическое, не хардкодится в HTML.** Счётчик и футер текущего теста считаются из `ORIGINAL_QUESTIONS.length` (`updateProgress`, `fillFooter` + склонение `pluralVoprosy`). Число вопросов в промо-кнопке перехода на *другой* тест берётся из `TEST_COUNTS` (`fillTransitionCount`, ключ — атрибут `data-count`). В разметке `<span class="tb-count">…</span>` — плейсхолдер, заполняется из JS.
- **Верхняя навигация, дата, дисклеймер.** В шапке обеих страниц — отдельная строка-навигация (`.top-nav`) со ссылкой на курс армянского языка (открывается в новой вкладке). В футере — «Последнее обновление: …» (`fillLastUpdated` из `LAST_UPDATED`). В фиолетовом боксе (`.section-intro`) на обеих страницах — дисклеймер о неофициальности тренажёра.
- **Блок статьи** под вопросом показывается только в режиме «Обучение» и только если у вопроса есть `article` и он есть в `ARTICLES`.
- **Лоадер** при входе в режим теста показывается только когда пул «свежий» (`testPoolFresh`: при первом заходе и после «Заново»), не на каждом переключении.
- **Проходной балл** — `Math.floor(questions.length/2)+1` (для теста = 17 из 33).
- Выбор режима сохраняется в `localStorage` (`MODE_KEY = 'arm-test-mode'`).

## Правила работы с контентом

- **Армянский — источник истины для теста.** Русский (`ru`) — вспомогательный перевод, в режиме «Самопроверка» скрывается CSS-классами (`body.test-mode .q-ru/.opt-ru/.ru-only`).
- Тексты вопросов/вариантов/правильных ответов сверяются с официальным источником: постановление № 1040-Н на arlis.am (`https://arlis.am/hy/acts/122105`). Тексты статей — Конституция РА 2015 на president.am.
- При смене формулировок отвлекающих вариантов приоритет у реальных экзаменационных, а не у выдуманных.
- **Нумерация статей приведена к редакции Конституции РА 2015.** `articles.js` содержит 165 ключей; тексты под номерами исправлены, поля `article` у вопросов согласованы. В частности, у official-вопросов поправлены 4 привязки: флаг → ст. 21, столица → ст. 22, административно-территориальные единицы → ст. 121, война → ст. 118.
- Правильность привязки `article` к вопросу проверять отдельно — там встречались ошибки.

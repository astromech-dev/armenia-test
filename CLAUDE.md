# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

Двуязычный (армянский + русский) веб-тест по Конституции Республики Армения для подготовки к экзамену на гражданство. Чистый статический сайт: HTML + CSS + ванильный JS, **без сборки, без npm, без фреймворков, без бэкенда**. Открывается как обычные файлы.

## Запуск

Локальный сервер (нужен именно сервер, а не `file://`, потому что страницы подключают внешние `.js`):

```bash
python3 -m http.server 8080
# затем открыть http://localhost:8080/ (базовый тест) и http://localhost:8080/full.html (расширенный)
```

Тестов/линтеров нет. Проверка — вручную в браузере (см. «Чек-лист поведения» ниже).

## Архитектура

Две страницы используют **одну общую логику и стили**, отличаясь только данными и разметкой шапки:

| Страница | Подключает | Банк вопросов |
|---|---|---|
| `index.html` (базовый тест) | styles.css, articles.js, **questions-official.js**, app.js | 33 официальных |
| `full.html` (расширенный тест) | styles.css, articles.js, **questions-full.js**, app.js | 175 (официальные + сгенерированные) |

Порядок подключения скриптов важен: данные (`articles.js`, `questions-*.js`) грузятся **до** `app.js`, который их читает через глобальные `const`.

**Контракт между файлами (всё через глобальные переменные, без модулей):**
- `articles.js` → `const ARTICLES` (карта `"номер_статьи" → русский текст`) и `const CONSTITUTION_URL`.
- `questions-official.js` / `questions-full.js` → `const ORIGINAL_QUESTIONS` (массив). Каждая страница грузит ровно один из них; имя переменной одинаковое, поэтому `app.js` не знает, какой тест запущен.
- `app.js` читает `ORIGINAL_QUESTIONS` и `ARTICLES`, рендерит в `#quiz`.

**Форма вопроса:**
```js
{ hy: "...", ru: "...",
  options: [ {hy,ru}, {hy,ru}, {hy,ru} ],
  correct: 0|1|2,            // индекс правильного варианта
  source: "official"|"generated",
  article: "47",             // ключ в ARTICLES; опционален
  note: "..." }              // служебная пометка происхождения; в UI не показывается
```

## Ключевые механизмы в app.js (поведение, которое нельзя ломать)

- **Два режима, независимое состояние.** `modeData.learn` и `modeData.test` — у каждого свой массив `questions` и свой `state` (answers/score/answered). Глобальные `questions`/`state` — ссылки на текущий режим.
- **Режим «Обучение»** = все вопросы (перетасованные). **Режим «Самопроверка»** = `TEST_QUESTION_COUNT` (33) случайных из всего банка (`buildTestQuestions` = shuffle + slice).
- **Бейджи источника автоматические:** `SHOW_BADGES = (число разных source) > 1`. Базовый тест (всё `official`) → бейджей нет; расширенный (смешанный) → бейджи «Официальный»/«Сгенерирован». Не хардкодить per-page.
- **Счётчик и футер считаются из `ORIGINAL_QUESTIONS.length`** (`updateProgress`, `fillFooter` + склонение `pluralVoprosy`). Не вписывать числа вручную в HTML.
- **Блок статьи** под вопросом показывается только в режиме «Обучение» и только если у вопроса есть `article` и он есть в `ARTICLES`.
- **Лоадер** при входе в режим теста показывается только когда пул «свежий» (`testPoolFresh`: при первом заходе и после «Заново»), не на каждом переключении.
- **Проходной балл** — `Math.floor(questions.length/2)+1` (для теста = 17 из 33).
- Выбор режима сохраняется в `localStorage` (`MODE_KEY = 'arm-test-mode'`).

## Правила работы с контентом

- **Армянский — источник истины для теста.** Русский (`ru`) — вспомогательный перевод, в режиме «Самопроверка» скрывается CSS-классами (`body.test-mode .q-ru/.opt-ru/.ru-only`).
- Тексты вопросов/вариантов/правильных ответов сверяются с официальным источником: постановление № 1040-Н на arlis.am (`https://arlis.am/hy/acts/122105`). Тексты статей — Конституция РА 2015 на president.am.
- При смене формулировок отвлекающих вариантов приоритет у реальных экзаменационных, а не у выдуманных.
- Правильность привязки `article` к вопросу проверять отдельно — там встречались ошибки (напр. админ-территориальные единицы — ст. 121, а не 9).
